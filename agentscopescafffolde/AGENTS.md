## What this project is

AgentScope is a Rust CLI tool — a scope firewall and audit layer for AI coding agents.
It wraps AI agents (Claude Code, Codex, Cursor, Gemini CLI) and enforces that they
only modify what the developer asked them to modify.

Core loop: `agentscope start "mission"` → run your agent → `agentscope check`

## File map

src/main.rs         — CLI entry point, routes subcommands
src/cli.rs          — Clap CLI definitions (Commands enum, AgentKind enum)
src/config.rs       — Config loading (agentscope.yaml), init, agent integration files
src/git.rs          — Git operations: baseline capture, working tree diff, commit history
src/policy.rs       — Policy engine: glob matching, IN SCOPE / UNASKED / BLOCKED logic
src/session.rs      — Session lifecycle: start, check, status, persistence to .agentscope/
src/judge.rs        — LLM judge: Ollama (primary), Claude, OpenAI providers
src/output.rs       — All non-TUI terminal output: Printer struct, CheckReport, Markdown
src/audit.rs        — `agentscope audit` command: activity log reading and display
src/tui.rs          — Ratatui TUI: watch mode, D-view (charts), slash commands

docs/TUI_VISUAL_SPEC.md   — COMPLETE colour, layout, and widget spec for the TUI.
                             Read this before touching tui.rs.
agentscope.yaml           — Default config file (generated by agentscope init)
README.md                 — Viral README (pain-first framing, do not genericise)

## Architecture rules

1. NEVER add unwrap() in production paths — use anyhow::Result and ? operator
2. ALL terminal output goes through src/output.rs Printer struct — no ad-hoc println
3. ALL TUI colours must match docs/TUI_VISUAL_SPEC.md exactly — no new colours
4. Session state lives in .agentscope/session.json (gitignored)
5. Activity log is append-only JSONL at .agentscope/activity.jsonl
6. Policy engine (policy.rs) is DETERMINISTIC — no AI calls, only globset matching
7. Judge (judge.rs) is OPTIONAL and async — always handle judge failure gracefully
8. Exit code 1 when BLOCKED files found (for CI). Exit code 0 otherwise.

## Cargo.toml notes

Key dependencies and why:
- clap 4.5 with derive — CLI parsing
- ratatui 0.28 + crossterm 0.28 — TUI (these versions must match)
- git2 0.19 — git operations (no shelling out to git)
- globset 0.4 — policy path matching (same crate ripgrep uses)
- ulid — session IDs (sortable, URL-safe, collision-resistant)
- serde + serde_yaml + serde_json — config and log serialization
- reqwest 0.12 async — LLM judge HTTP calls
- tokio 1 full — async runtime
- console + owo-colors — non-TUI terminal styling

## Colour rules for agents (CRITICAL)

DO NOT use Color::White, Color::Yellow, Color::Blue as named colours.
ALWAYS use Color::Rgb(r, g, b) from the constants in TUI_VISUAL_SPEC.md.
Named colours render differently across terminals. Rgb is exact.

The 8 most-used colours (memorise these):
  CLR_PURPLE  = Color::Rgb(160, 120, 255)  — agentscope brand, section headers
  CLR_CYAN    = Color::Rgb(80,  210, 220)  — session IDs, cursor, interactive
  CLR_GREEN   = Color::Rgb(80,  200, 120)  — IN SCOPE, success, live dot
  CLR_AMBER   = Color::Rgb(230, 160, 50)   — UNASKED, warnings, health bar
  CLR_RED     = Color::Rgb(220, 80,  80)   — BLOCKED, errors, drift
  CLR_TEAL    = Color::Rgb(60,  180, 160)  — agent names (CLAUDE, CODEX)
  CLR_TEXT_PRI= Color::Rgb(220, 225, 235)  — primary readable text
  CLR_TEXT_DIM= Color::Rgb(72,  78,  95)   — hints, decorative, dim text

## Testing approach

cargo test              — unit tests
cargo test --test '*'   — integration tests (uses assert_cmd + tempfile)

Each integration test should:
1. Create a tempdir with git init + initial commit
2. Write an agentscope.yaml
3. Run agentscope start "mission"
4. Modify some files (including some that should be BLOCKED)
5. Run agentscope check and assert exit code + output contains expected strings

## Build & run

cargo build                    # debug build
cargo build --release          # release (stripped, LTO, ~3MB binary)
cargo run -- start "test"      # run with args
cargo run -- check             # check active session
cargo run -- watch             # launch TUI
